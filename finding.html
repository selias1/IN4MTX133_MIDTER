Summary of Findings

I felt that User tests and Cognitive Walkthroughs were the most useful aspects of User Testing. For Cognitive Walkthroughs I was able to observe users navigate through my web page and see if there were any aspects to my website that they felt was unintuitive. Fortunately, most of my users did not have any issues with the way how my website was designed since they were able to find everything that they wanted to see.



User tests were extremely useful because it provided much needed feedback that I needed to cover some aspects to web design that I did not account for. Most people generally found my skills page to be useful and a very effective way to gauge my technical abilities. But, there were few people that felt that it was not too effective. One user mentioned that the way how my webpage was designed did not provide enough information. I had bars that were colored in to show my skill level, but to them, it was just simple bars that were meaningless. Another user mentioned that bars were not an effective means to showcase my skill level because they felt that it was too subjective. Instead, they offered a different solution, and that solution was to provide links to my personal projects and let the user decide the level of skill for themselves.



Incorporation of Findings

After much thought and consideration, I decided to leave the design of my technical skills section the way it is. The purpose of that section was to show the user of the website a general idea of how I gauge MYSELF and how confident I feel about my skill level in that particular language. Furthermore, I had an interviewer ask me how I would rate myself and I feel that the current design best answers that question. However, I do plan on changing it up a bit. Rather than showing bars, I think I am going to put a bar split into ten pieces and color individual pieces in to better quantify my skill level.



I also plan on adding a github icon at the very bottom of my webpage along with my linkedin and facebook icon. This is in response to my lack of showing my personal projects. However, I feel that it is not good enough to just provide links to my github account because it requires the user to manually download and build the project to see it in action. Instead, I plan on bettering my skills as a web developer to showcase my projects by directly linking it to a webpage, which does not require any part in the user and show the fruits of my successes in web development.
